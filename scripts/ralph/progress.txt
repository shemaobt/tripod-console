## Codebase Patterns
- Backend repo: /workspace/tripod-backend/ — Python deps NOT installed in devcontainer, can only verify syntax (ast.parse)
- Backend migrations: revision ID format `YYYYMMDD_NNNN`, chain via `down_revision` to latest existing
- Build: `npm run build` (tsc -b && vite build), `npm run typecheck` (tsc -b), `npm run lint` (eslint)
- Path alias: `@/` maps to `src/` (configured in tsconfig.app.json and vite.config.ts)
- Tailwind CSS v4 uses `@import "tailwindcss"` in index.css (not `@tailwind` directives)
- Google Fonts `@import url()` must come BEFORE `@import "tailwindcss"` to avoid CSS warning
- Design tokens defined in `@theme {}` block in index.css — use `--color-*` naming for Tailwind colors
- Style constants in `src/styles/` — import from `@/styles` (e.g., `import { card } from "@/styles"`)
- Use `cn()` from `@/utils/cn` for all class merging (clsx + tailwind-merge)
- UI components in `src/components/ui/` follow shadcn pattern: Radix + Tailwind + cva + cn()
- ESLint `react-refresh/only-export-components` is disabled for `src/components/ui/` (exports variants alongside components)
- Use `type` alias instead of empty `interface extends` to avoid `@typescript-eslint/no-empty-object-type` lint error
- API proxy: `/api` -> `http://localhost:8000` in vite dev server
- React 18 (not 19) — use `react` and `react-dom` ^18.3.1 with `@types/react` ^18.3.12
- ESLint `react-hooks/set-state-in-effect`: don't call setState in useEffect; use `useSyncExternalStore` or `useMemo` for derived state
- Context files exporting Provider + hook: add `// eslint-disable-next-line react-refresh/only-export-components` above the hook export
- AuthProvider must be inside BrowserRouter (uses useNavigate); useAuth() for accessing auth state
- Auth types: User, MyRoleResponse from `@/types`; token keys from `@/constants/app`
- Routing: BrowserRouter in App.tsx, `/app` shell uses Outlet, pages in `components/pages/`
- Common components in `src/components/common/`: LoadingSpinner, EmptyState, ConfirmDialog, ErrorBoundary, InfoTooltip, FeatureSpotlight
- Zustand stores in `src/stores/` — onboardingStore persists to localStorage with key `tc_onboarding`
- ErrorBoundary is a class component (React requirement for error boundaries)
- Types in `src/types/` — import from `@/types` (re-exported via index.ts)
- API client in `src/services/api.ts` — namespaced exports: authAPI, usersAPI, appsAPI, languagesAPI, orgsAPI, projectsAPI, rolesAPI
- Token keys in `src/constants/app.ts`: ACCESS_TOKEN_KEY, REFRESH_TOKEN_KEY
- All API methods typed with AxiosPromise<T> generics; datetime fields are `string` in TypeScript
- AdminRoute guard in App.tsx: wraps admin-only routes, checks `isPlatformAdmin`, renders AccessDeniedPage if false
- Pages inside AppShell use flex-1 layout (not min-h-screen) since they render inside the shell
- Backend service pattern: one function per file in `app/services/<domain>/`, `__init__.py` re-exports; add alias in `app/services/__init__.py`
- Backend admin endpoints use `Depends(require_platform_admin)`, auth endpoints use `Depends(get_current_user)` from `app/core/auth_middleware`
- Backend Pydantic schemas with ORM conversion need `model_config = {"from_attributes": True}`
- Backend services raise business exceptions (NotFoundError, ConflictError) — never HTTPException
- When adding new service+router: update `app/services/__init__.py` (import + alias) AND `app/main.py` (include_router)
- FastAPI route ordering: fixed path routes (e.g. `/my-apps`) must be defined BEFORE parameterized routes (e.g. `/{app_id}`)
